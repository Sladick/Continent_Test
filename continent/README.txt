1. Дана модель предметной области с сущностями:
- Страница
- Контент типа видео:
    - специфичные атрибуты: ссылка на видео, ссылка на файл с субтитрами
- Контент типа аудио:
    - специфичные атрибуты: битрейт в количестве бит в секунду.
- Контент типа текст:
    - специфичные атрибуты: поле для хранения текста произвольной длинны

2. Нужно учитывать, что специфичные атрибуты разных видов контента существенно различаются.
3. У всех видов контента должен присуствовать атрибут "счетчик просмотров".
4. У всех видов контента есть атрибут "Заголовок".
5. Со страницей может быть связан любой вид контента в любом колличестве .

            ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ:

1. Сделать API для получения списка всех страниц.
    - Должна поддерживаться пагинация
    - В ответе должен содержаться URL на API с детальной информацией о странице.
2. Сделать API для получения детальной информации о странице.
    - помимо атрибутов страницы в ответе должны содержаться все привязанные к странице объекты контента в виде
    вложенной структуры.
3. При обращении к API с деталями о странице счетчик просмотров каждого объекта контента , привязанного к странице
 должен увеличиваться на единицу.
4. Заведение страниц и привязка к ним контента должна выполняться через админку.
    - должен поддерживаться поиск по заголовку.
    - желательно для удобства реализовать привязку и упраление контантом на странице в виде inline-блоков в разделе
    управления страницей.
